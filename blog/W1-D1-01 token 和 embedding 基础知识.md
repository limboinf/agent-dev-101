# Token 和 Embedding 基础（面向大语言 LLM 入门教程）

---

## 一、Token：大语言模型的输入基础

我们日常输入给 LLM 的文本无法直接被模型处理，模型需要先把文本拆分成一个个最小处理单元，这就是**Token**。

常见的拆分逻辑：

1. 常见的词语会被拆为1个Token，比如`langchain`作为一个完整专有名词会被分成1个Token
2. 生僻词、未登录词会被拆分为多个子词Token，避免无法识别的问题
3. 中文字符一般大约1个字对应1~1.5个Token，不同分词算法会略有差异

你可以把Token理解为大模型的"识字单位"，我们常说的上下文窗口大小，本质就是指模型单次能处理的最大Token数量。

## 二、Embedding：让文本变成模型能读懂的向量

拿到Token之后，需要把每个Token转换为计算机能计算的数值形式，这就是**Embedding（词嵌入）**：

简单来说，就是把每个Token映射为一个固定长度的浮点数向量，语义相似的Token，对应的向量在向量空间中的距离也更近。

比如`大语言模型`和`LLM`这两个表述语义相近，它们的Embedding向量的余弦相似度就会远高于和`猫`的向量相似度。这种特性也让Embedding广泛应用在检索、RAG、语义匹配等场景中。

## 三、和当前主流技术栈的关联

在我们搭建基于LLM的应用，比如用langchain、langgraph开发agent时，Token和Embedding是最基础的核心：

1. Token计数直接关联我们调用大模型API的成本，也决定了能塞入上下文的信息总量
2. Embedding的质量直接决定了RAG检索、agent工具匹配这类任务的效果，想要优化应用效果，很多时候都要从Embedding模型选型和调优入手。

你可以在github上找到各类开源的分词工具、Embedding模型来做实践测试，是入门大语言应用开发非常好的起点。
